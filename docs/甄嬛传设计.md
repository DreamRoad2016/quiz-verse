# 《甄嬛传》猜人物 — 设计与录入

> **题包路径**：[`src/main/resources/packs/zhenhuan_2011/`](../src/main/resources/packs/zhenhuan_2011/)  
> **文件**：`pack.yaml` + `schema.yaml` + `entities.json`  
> **玩法页**：`/guess.html?pack=zhenhuan_2011`

---

## 1. 存储约定

| 层 | 做什么 |
|----|--------|
| `entities.json` | 每个角色的 `name` / `aliases` / `attrs` |
| `schema.yaml` | 比对列、枚举选项、`enumRef` |
| `pack.yaml` | 标题、最大猜测次数等 |

改选项或加角色：编辑上述文件后重启应用。

---

## 2. 称呼 ≠ 位分（重要）

| 概念 | 存哪 | 例子 | 参与比对？ |
|------|------|------|------------|
| 姓名 | `name` | 甄嬛 | 否（选人） |
| 称呼 / 别名 | `aliases[]` | 嬛嬛、莞莞、熹贵妃 | 否（联想） |
| 位分 | `attrs.titles[]` | 常在、贵人、嫔、妃 | **是** |

「熹贵妃」作称呼进 `aliases`；作册封进 `titles`。不要混在一个字段。

---

## 3. 字段一览

| 字段 | 形态 | 比对 |
|------|------|------|
| gender | 单值 | exact |
| role_type | 单值 | exact |
| titles | 多值 | set |
| residences | 多值 | set |
| major_plots | 多值（存 key） | set + enumRef |
| connections | 多值（存其他角色 id） | set（展示解析为姓名） |

枚举写在 `schema.yaml` 的 `enums` 下。

---

## 4. 实体示例

```json
{
  "id": "zh_huan",
  "name": "甄嬛",
  "aliases": ["嬛嬛", "莞莞", "熹贵妃"],
  "attrs": {
    "gender": "女",
    "role_type": "后宫嫔妃",
    "titles": ["常在", "贵人", "嫔", "妃", "贵妃", "皇贵妃", "太后"],
    "residences": ["碎玉轩", "永寿宫"],
    "major_plots": ["chuxuan_dianxuan", "jinghong_wu", "dixue_qinzi"],
    "connections": ["zh_yixiu", "zh_huafei", "zh_emperor"]
  }
}
```

---

## 5. 怎么加角色

1. 在 `entities.json` 追加一条（`id` 唯一）
2. `attrs` 键与 schema 一致；剧情用 key；关系用其他角色 id
3. 若有新枚举值，先改 `schema.yaml` 的 `enums`
4. 重启 → 打开猜局页验证联想与比对

批量生产可用 [题包生成提示词](./08_猜人物场景_题包生成提示词.md)。
