# 《甄嬛传》猜人物 — 设计与录入

> **字段选项（可编辑）**：[`config/works/zhenhuan_2011/`](../config/works/zhenhuan_2011/)  
> **库表**：`work` / `work_character` / `work_column`  
> **存储**：玩法字段在 **`attrs` jsonb**；录入页读本地 json 渲染单选/多选

---

## 1. 为什么还用 attrs？

| 层 | 做什么 |
|----|--------|
| **PostgreSQL `attrs`** | 存每个角色**已选中的值**（如 `titles: ["贵人","妃"]`） |
| **`config/works/.../fields.json`** | 定义有哪些字段、单选还是多选、是否参与比对 |
| **`config/works/.../enums/*.json`** | 下拉/多选的**可选项**，改文件即可增删，不必改表 |
| **录入页（待做）** | 读 config 画表单 → 保存进 `call_names` + `attrs` |

attrs 只是**存储格式**，不是「写死在代码里的字段」。你要的「可编辑、枚举可选、单选多选」由 **config + 录入页** 解决。

**重大剧情也可以进 attrs**：`major_plots: ["dixue_qinzi","jinghong_wu"]`，选项在 `enums/major_plots.json`。不必单独剧情表才能玩；以后剧情特别多再考虑拆表。

---

## 2. 称呼 ≠ 位分（重要）

| 概念 | 存哪 | 例子 | 参与猜局比对？ |
|------|------|------|----------------|
| **姓名** | `display_name` | 甄嬛 | 否（用于选中谁） |
| **称呼** | `call_names[]` | 嬛嬛、莞莞、熹贵妃（别人怎么叫） | **否**，只联想搜索 |
| **位分** | `attrs.titles[]` | 常在、贵人、嫔、妃 | **是** |

「熹贵妃」作称呼可以进 `call_names`；作册封位分进 `titles`。两者不要混在一个字段里。

---

## 3. 字段一览（MVP）

| 字段 | 单选/多选 | 选项文件 | 在 attrs？ |
|------|-----------|----------|------------|
| 称呼 | 多标签 | —（自由填或从常用称呼补） | 否，列 `call_names` |
| 性别 | 单选 | `enums/gender.json` | 是 |
| 身份类别 | 单选 | `enums/role_type.json` | 是 |
| 位分 | 多选 | `enums/titles.json` | 是 |
| 住所 | 多选 | `enums/residences.json` | 是 |
| 重大剧情 | 多选 | `enums/major_plots.json`（key+label） | 是 |
| 关系网 | 多选 | 选其他角色 | 是，`connections: [id…]` |

改玩法：编辑 `config` 里 json → 录入页自动出新选项；已有角色数据仍在 `attrs` 里，可慢慢改。

## 4. attrs 示例

```json
{
  "gender": "女",
  "role_type": "后宫嫔妃",
  "titles": ["常在", "贵人", "嫔", "妃", "贵妃"],
  "residences": ["碎玉轩", "永寿宫"],
  "major_plots": ["jinghong_wu", "dixue_qinzi"],
  "connections": [2, 3]
}
```

## 5. 怎么开始

### 5.1 建库

```bash
psql -d quiz_verse -f db/work_guess_schema.sql
psql -d quiz_verse -f db/seeds/zhenhuan_2011_seed.sql
```

旧库升级：`psql -d quiz_verse -f db/migrations/003_attrs_config_v3.sql` 后再跑 seed。

### 5.2 先改选项（不用动数据库）

编辑例如：

- `config/works/zhenhuan_2011/enums/titles.json` — 加/删位分  
- `config/works/zhenhuan_2011/enums/major_plots.json` — 加一场新剧情  

### 5.3 录角色（SQL 暂时代替录入页）

```sql
-- 第一步：只建人
INSERT INTO work_character (work_id, display_name, call_names)
VALUES ('zhenhuan_2011', '甄嬛', ARRAY['嬛嬛','莞莞','熹贵妃']);

-- 第二步：补 attrs 并发布
UPDATE work_character SET
  attrs = '{
    "gender":"女",
    "role_type":"后宫嫔妃",
    "titles":["常在","贵人","嫔","妃","贵妃"],
    "residences":["碎玉轩","永寿宫"],
    "major_plots":["jinghong_wu","dixue_qinzi"],
    "connections":[]
  }'::jsonb,
  status = 'ready',
  is_active = true
WHERE work_id = 'zhenhuan_2011' AND display_name = '甄嬛';
```

`draft` 状态可只录姓名；`ready` + `is_active=true` 才进猜局题库。

### 5.4 批量给一场戏「挂人」（仍用 attrs）

在录入页做好之前：对每个出场角色，把该剧情 key **追加**进 `major_plots` 数组即可（不必单独剧情表）。

### 5.5 下一步开发（还没做）

1. **录入页** `work-edit.html?workId=zhenhuan_2011`：读 `fields.json` + enums，保存角色  
2. **猜谜页** `guess-work.html?workId=zhenhuan_2011`：读 `work_column` 比对  
3. 选项管理小页：直接编辑 enums json 或表单写回文件  

---

## 6. 不建议

- 演员、出场集数、最终位分/生死（剧透）  
- 把位分写进 `call_names`  
- 还没开始玩就拆「一剧一张表」

---

## 7. 设计原则

好玩来自 **名场面 + 位分 + 住所 + 关系**，不是百科全书。  
**config 管选项，attrs 管取值，录入页管编辑**——三者分开，才能既简单又可改。
