# 《甄嬛传》猜人物游戏字段设计（MVP）

## 设计目标

《甄嬛传》适合做“关系推理型”猜人物游戏。

玩家对角色的记忆，主要来自：

- 身份与位分
- 所属人群
- 宫殿与住所
- 人际关系网

因此字段设计应优先服务：

- 推理感
- 信息量
- 半匹配反馈（黄）
- 角色辨识度

而不是百科资料化。

---

# 核心字段

## 1. 性别

```text
field: gender
type: enum
```

推荐值：

```text
男
女
```

用途：

- 快速缩小范围
- 区分后妃 / 太监 / 官员 / 皇族等

比对：

```text
相同 = 绿
不同 = 灰
```

---

## 2. 身份 / 位分（核心字段）

```text
field: titles
type: text[]
```

存储角色经历过的所有身份。

示例：

```json
[
  "常在",
  "贵人",
  "嫔",
  "妃",
  "贵妃"
]
```

或：

```json
[
  "太监",
  "总管太监"
]
```

设计原因：

- 《甄嬛传》的核心结构就是等级制度
- 多值可以形成“部分匹配”
- 既能提示，也能迷惑

推荐比对规则：

```text
完全一致 = 绿
有交集 = 黄
无交集 = 灰
```

---

## 3. 住所 / 宫殿

```text
field: residences
type: text[]
```

存储角色长期或重要时期的住所。

示例：

```json
[
  "碎玉轩",
  "永寿宫"
]
```

设计原因：

- 宫殿本身就是角色标签
- 玩家记忆度非常高
- 适合产生关联推理

推荐比对规则：

```text
完全一致 = 绿
有交集 = 黄
无交集 = 灰
```

---

## 4. 身份类别

```text
field: role_type
type: enum
```

推荐值：

```text
后宫嫔妃
皇族
宫女
太监
太医
侍卫
前朝官员
嬷嬷
```

设计原因：

- titles 太细
- role_type 提供“大类推理”

例如：

```text
贵人 / 嫔 / 妃
都属于：
后宫嫔妃
```

比对：

```text
相同 = 绿
不同 = 灰
```

---

## 5. 人际关系网（推荐）

```text
field: connections
type: bigint[]
```

存储关联角色 ID。

示例：

```json
[1001, 1002, 1003]
```

对应：

```text
甄嬛
皇帝
佩儿
```

设计原因：

《甄嬛传》的核心是人物关系，而不是单纯阵营。

该字段可以提供：

- 弱关联推理
- 人物网络感
- 高娱乐性

推荐比对规则：

```text
完全一致 = 绿
有交集 = 黄
无交集 = 灰
```

注意：

不要直接存名字：

```json
["皇上", "四郎", "皇帝"]
```

应统一存角色 ID。

---

# 推荐补充字段

## aliases（别名）

```text
field: aliases
type: text[]
```

示例：

```json
[
  "莞嫔",
  "熹贵妃"
]
```

用途：

- 搜索联想
- 前端 autocomplete

不参与比对。

---

# 不建议做的字段

## 阵营 / 派系

原因：

- 《甄嬛传》核心是关系网
- faction 不如 connections 自然

---

## 演员

原因：

- 会变成“猜演员”
- 不是角色信息

---

## 出场集数

原因：

- 没有推理价值
- 属于冷知识

---

## 最终位分

原因：

- 剧透严重

---

## 生死状态

原因：

- 剧透严重

建议：

- 困难模式再开放

---

# 推荐最终字段（MVP）


| 字段          | 类型       | 必须  |
| ----------- | -------- | --- |
| gender      | enum     | 是   |
| role_type   | enum     | 是   |
| titles      | text[]   | 是   |
| residences  | text[]   | 是   |
| connections | bigint[] | 是   |
| aliases     | text[]   | 推荐  |


---

# 核心设计思想

这个游戏的重点不是“百科全书”。

而是：

```text
玩家如何通过角色特征进行推理。
```

《甄嬛传》的玩家记忆核心是：

- 她是什么身份
- 她住哪宫
- 她跟谁关系近
- 她属于什么人群

因此：

“关系网 + 位分 + 宫殿”

才是最适合该作品的猜人物维度。
