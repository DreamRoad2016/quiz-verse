# 影视剧猜人物 — 一步步操作手册（测试环境录入）

> **录入页面**（测试用）：[`/work-studio.html`](http://localhost:8098/work-studio.html) → [`/work-character-studio.html`](http://localhost:8098/work-character-studio.html)  
> **整表导入导出**（上线/备份）：[`/db-admin.html`](http://localhost:8098/db-admin.html)  
> **管理密钥**：`application.yml` 中 `quiz.admin.api-key`（默认开发密钥见配置文件）

---

## 第 0 步：环境准备

### 0.1 启动 Redis

```bash
brew services start redis
# 或：docker run -d --name quiz-verse-redis -p 6379:6379 redis:7-alpine
```

### 0.2 创建数据库（仅首次）

```bash
createdb quiz_verse
```

### 0.3 建表（按顺序执行）

在**项目根目录** `quiz-verse/` 下：

```bash
# ① 影视剧通用表（work / work_character / work_column）
psql -d quiz_verse -f db/work_guess_schema.sql

# ② 甄嬛传作品元数据（比对列 + 可选，已有则跳过）
psql -d quiz_verse -f db/seeds/zhenhuan_2011_seed.sql
```

**会创建哪些表？**

| 表名 | 作用 |
|------|------|
| `work` | 一部影视剧一行（如 `zhenhuan_2011`） |
| `work_column` | 该作品的猜局比对列定义 |
| `work_character` | 所有作品的角色（用 `work_id` 区分） |

（LoL Demo 独立：`psql -d quiz_verse -f db/demo_lol_schema.sql`，与影视剧无关。）

**若之前跑过旧版库**：

```bash
psql -d quiz_verse -f db/migrations/003_attrs_config_v3.sql
psql -d quiz_verse -f db/seeds/zhenhuan_2011_seed.sql
```

### 0.4 校验

```bash
psql -d quiz_verse -c "SELECT id, title_cn FROM work;"
psql -d quiz_verse -c "\dt work*"
```

期望至少看到一行 `zhenhuan_2011 | 甄嬛传`。

### 0.5 启动后端

```bash
mvn spring-boot:run
```

浏览器可访问：**http://localhost:8098**

---

## 第 1 步：注册影视剧（页面 ①）

1. 打开 **http://localhost:8098/work-studio.html**
2. 输入 **X-Admin-Key**（与 `application.yml` 里 `quiz.admin.api-key` 一致）
3. 若种子已执行，列表里应有 **甄嬛传 / zhenhuan_2011**
4. 若要新建另一部剧（测试）：
   - **作品 ID**：小写英文+下划线，如 `demo_drama_01`
   - **中文名**：随意
   - **模板**：选 `甄嬛传模板`（会复制 `config/works/zhenhuan_2011/` 到新目录，并复制 `work_column`）
5. 点某作品右侧 **「录入角色」** → 进入页面 ②

---

## 第 2 步：录入角色（页面 ②）

1. 地址形如：**http://localhost:8098/work-character-studio.html?workId=zhenhuan_2011**
2. 左侧 **「+ 新建角色」**
3. 填写：
   - **姓名**：规范名（如 `甄嬛`）
   - **称呼**：嬛嬛、莞莞（**不是位分**）
   - **性别 / 身份类别**：单选下拉（选项来自 `config/works/zhenhuan_2011/enums/`）
   - **位分 / 住所 / 重大剧情**：多选勾选
   - **关系网**：勾选其他已保存的角色
4. **录入状态**：
   - `draft`：可先只填姓名，稍后补全
   - `ready`：属性填够后再选
5. **上架猜局题库**：选「是」时须 `ready`（猜谜 API 做好后才会真正进局）
6. 点 **保存**；可继续新建或点左侧列表编辑

### 改下拉选项（不用动数据库）

编辑本地文件，保存后**刷新角色页**即可看到新选项：

```
config/works/zhenhuan_2011/enums/titles.json      # 位分
config/works/zhenhuan_2011/enums/residences.json  # 住所
config/works/zhenhuan_2011/enums/major_plots.json # 重大剧情
```

---

## 第 3 步：备份 / 批量迁移（上线用 Data Studio）

1. 打开 **http://localhost:8098/db-admin.html**
2. 同一 **X-Admin-Key**
3. 左侧选表：`work` / `work_character` / `work_column`
4. **导出 JSON** / **粘贴 JSON 数组导入**（按主键 upsert）

白名单已在 `application.yml` 的 `quiz.admin.tables` 中配置。

---

## 推荐录入顺序（甄嬛传）

1. 先录 **10～20 个核心人物**（可全 `draft`，只填姓名+称呼）
2. 再逐个补 **性别、位分、住所**
3. 补 **重大剧情**（多选名场面）
4. 最后勾 **关系网**（依赖其他人已存在）
5. 满意后改 `ready` + 上架

---

## 常见问题

| 问题 | 处理 |
|------|------|
| 页面 401 | 检查 X-Admin-Key |
| 角色页无下拉 | 确认 `config/works/{workId}/fields.json` 存在；新建作品是否复制模板成功 |
| 保存失败 connections | 引用的角色 id 必须已存在；可先留空 `[]` |
| 数据库连接失败 | 检查 `application.yml` 的 `spring.datasource.*` 与 PostgreSQL 是否启动 |
