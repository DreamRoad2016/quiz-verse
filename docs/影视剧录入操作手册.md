# 影视剧类题包 — 维护手册

适用于「一部作品一个 pack」的猜人物题库（如甄嬛传）。

## 1. 目录

```
src/main/resources/packs/{work_id}/
  pack.yaml
  schema.yaml
  entities.json
```

`work_id` 建议稳定、可 URL，例如 `zhenhuan_2011`。

## 2. 新建一部剧

1. 复制 `zhenhuan_2011` 目录并改名
2. 改 `pack.yaml` 的 id / title / description
3. 按该剧设定改 `schema.yaml` 的列与 enums
4. 清空并填写 `entities.json`
5. 重启应用，打开 `/guess.html?pack={work_id}`

## 3. 日常改数据

| 需求 | 改哪里 |
|------|--------|
| 加/改角色 | `entities.json` |
| 加位分/住所等选项 | `schema.yaml` → `enums` |
| 加比对列 | `schema.yaml` → `columns`，并给已有角色补 attrs |
| 改最大次数 | `pack.yaml` → `maxGuesses` |

无需建库、无需管理后台。用 Git 做 diff 与回滚。

## 4. 质量检查（建议）

- 每个 entity 有唯一 `id`、非空 `name`
- `attrs` 中的枚举值落在 schema enums 内（剧情用 key）
- `connections` 里的 id 都能在本包 entities 中找到
- 启动后 `/api/packs/{id}/briefs` 条数符合预期
- 手动猜 2～3 局，确认绿/黄/灰符合直觉

## 5. 与旧方案的区别

旧方案（已废弃）：PostgreSQL `work` / `work_character` + `config/works/*/fields.json` + studio 页。  
现方案：单一题包目录，编辑文件即维护。远期若上 PG，可从 `entities.json` 导入。
