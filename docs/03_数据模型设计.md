## 3.1 MVP：题包（pack）

每个可玩题材一个目录：`src/main/resources/packs/{packId}/`

| 文件 | 作用 |
|------|------|
| `pack.yaml` | id、title、description、maxGuesses、kind（demo/official）、tags |
| `schema.yaml` | 比对列定义 + 枚举 |
| `entities.json` | 人物数组 |

### 实体形状

```json
{
  "id": "zh_huan",
  "name": "甄嬛",
  "aliases": ["嬛嬛", "莞莞", "熹贵妃"],
  "attrs": {
    "gender": "女",
    "titles": ["常在", "贵人", "嫔", "妃"],
    "major_plots": ["jinghong_wu", "dixue_qinzi"]
  }
}
```

- `name` + `aliases` → brief（联想）
- `attrs` → 仅服务端比对；键名与 `schema.yaml` 中列的 `attr` 对齐

### 列类型（schema.columns[].type）

| type | 含义 |
|------|------|
| `identity` | 同一实体才绿（展示姓名） |
| `exact` | 单值全等 |
| `set` | 集合：全等绿 / 相交黄 / 否则灰 |
| `number` | 数值；可配 `nearThreshold`，带 ↑↓ |

可选 `enumRef`：把存值映射成展示文案（如剧情 key → 中文名）。关系类字段可存其他实体 id，展示时解析为姓名。

### 与「大类 / 标签」的关系

- **当前**：一个 pack ≈ 一个大类或一部作品池（如甄嬛传）；开局选 pack。
- **远期**（ADR-001）：大类下多标签并集筛选，可在 pack 内加 `tags` 字段，或升级到 PG 后再做。

## 3.2 运行时对象

```
PackRegistry
  └─ LoadedPack
       ├─ PackMeta
       ├─ PackSchema
       └─ List<PackEntity>

MatchSession（内存或 Redis）
  matchId / packId / answerId / guessCount / state
```

终局前答案 id 只存在服务端 session 中。
