# 6. 技术栈与架构

> 遵循 [ADR-002](./DECISIONS.md)、[ADR-003](./DECISIONS.md)、[ADR-007](./DECISIONS.md)、[ADR-008](./DECISIONS.md)。对战规划见 [ADR-006](./DECISIONS.md) 与 [05](./05_对战模式设计.md)。

## 6.1 当前架构（已实现）

```
┌─────────────────────────┐          ┌──────────────────────────────────┐
│  浏览器静态页             │  HTTP    │  Spring Boot 3 (Java 17)          │
│  index / guess.html      │ ───────► │  GuessApiController               │
└─────────────────────────┘          │  PackLoader → PackRegistry        │
                                     │  CompareEngine                    │
                                     │  MatchService                     │
                                     │       │                           │
                                     │       ▼                           │
                                     │  MatchStore: memory | redis       │
                                     │  Packs: classpath:/packs/*        │
                                     └──────────────────────────────────┘
```

- **无 PostgreSQL**（MVP）
- **无录入后台**；改题包文件即可
- **无 WebSocket**

## 6.2 技术选型

| 层 | 选型 |
| --- | --- |
| 运行时 | Java 17 + Spring Boot 3.3 |
| 内容 | 题包 YAML/JSON |
| 对局 | 默认内存；`QUIZ_MATCH_STORE=redis` 可选 |
| 验证前端 | HTML + Vue 3（CDN） |
| 构建 | Maven |

## 6.3 阶段化路线

```
已完成：题包引擎 + LoL Demo + 甄嬛传包 + 静态猜局页
    ↓
下一阶段：扩题包 / 热加载 / 更强联想；按需引入 Redis 默认化
    ↓
对战（ADR-006）：Redis 房间 + REST 短轮询
    ↓
持久化：PostgreSQL（账号、战绩、多人录入）
    ↓
上云 + 小程序换皮（接口尽量保持）
```

## 6.4 关键 API（当前）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/packs` | 题包列表 |
| GET | `/api/packs/{id}/briefs` | brief 列表 |
| POST | `/api/matches` | `{ packId }` 开局 |
| POST | `/api/matches/{id}/guess` | `{ entityId }` 猜测 |

## 6.5 安全要点

- attrs 与答案 id 不下发（ADR-002）
- 管理类接口若未来恢复，必须独立鉴权；勿把开发用密钥写进公开文档正文示例以外的生产配置
