# 4. 单局猜测算法

> - 所有比对在服务端（[ADR-002](./DECISIONS.md)）  
> - 当前实现按 **题包 schema** 读 attrs（[ADR-007](./DECISIONS.md)）  
> - 远期大类子表方案见 [03 附录](./03_数据模型设计.md)

## 4.1 总体流程（当前 API）

```
[客户端]                                           [服务端]

GET /api/packs/{id}/briefs                    →  briefList（无 attrs）
POST /api/matches { packId }                  →  随机 answerId 写入 MatchSession
                                                  返回 matchId + columns + maxGuesses

POST /api/matches/{id}/guess { entityId }     →  读 guess / answer 实体
                                                  按 schema 逐列比对
                                                  返回 display + cells + gameStatus
                                                  命中或耗尽时附带 answerReveal（brief）
```

## 4.2 列比对规则

| type | 绿 | 黄 | 灰 | 附带 |
|------|----|----|----|------|
| identity | 同一 id | — | 否则 | — |
| exact | 值相等 | — | 否则 | — |
| set | 集合相等 | 非空相交 | 否则 | matched 可选 |
| number | 相等 | \|Δ\| ≤ nearThreshold | 否则 | arrow ↑/↓ |

空值或缺省：按 `unknown` / 灰处理（实现以 `CompareEngine` 为准）。

## 4.3 单元格结果形状

```json
{
  "kind": "exact | partial | none | near | higher | lower | unknown",
  "color": "green | yellow | gray",
  "arrow": "↑ | ↓ | null",
  "matched": ["可选", "相交项"]
}
```

客户端用 `display[key]` 展示猜测侧文案，用 `cells[key]` 着色。

## 4.4 终局

- `hit`：guessId == answerId → `WON`，揭示答案 brief  
- 次数用尽且未命中 → `LOST`，揭示答案 brief  
- 进行中不返回答案 id

## 4.5 与远期「大类 + 标签」的差异

规划中的「选大类 + 标签并集筛库」会在开局参数中增加过滤条件；**逐列比对规则不变**，仍由该局锁定的 schema 驱动。
