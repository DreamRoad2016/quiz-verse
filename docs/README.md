## 核心决策一览

| ID | 决策 | 一句话 |
| --- | --- | --- |
| **ADR-001** | 题库组织 | 每局锁 1 个大类/题包；大类内标签并集（远期） |
| **ADR-002** | 比对位置 | **服务端比对**；客户端只有 brief |
| **ADR-003** | 后端形态 | 必须有独立后端 |
| **ADR-004** | 多分类 | 同一人物可归多个大类（远期 PG 方案） |
| **ADR-005** | 技术栈（历史） | 曾规划 Java + PG + Redis；见 ADR-007/008 修订 |
| **ADR-006** | 对战 | REST 短轮询 + Redis（规划中） |
| **ADR-007** | 内容源 | **MVP 以题包文件为真相源**，不先上 PG/录入台 |
| **ADR-008** | 运行时 | **Java 17 + Spring Boot 3**；对局默认内存，Redis 可选 |

## 一句话总结

> 做一款类 LoLdle / Wordle 的猜人物产品：按题包组织内容与比对列，服务端裁判，本地 HTML 验证玩法；后续再上持久化、对战与小程序。
