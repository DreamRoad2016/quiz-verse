# 7. 里程碑与风险

> 路线总纲：题包引擎验证玩法 → 扩内容 → 对战 → 持久化 / 上云 → 小程序。

## 7.1 里程碑

### 已完成 — 干净主干重写

- Java 17 + Spring Boot 3 题包引擎（加载 / 比对 / 对局）
- 题包：`lol_cn_kr`、`zhenhuan_2011`
- 首页与猜局页（服务端比对、终局反馈）
- 文档对齐 ADR-007 / ADR-008

### M1 — 内容与体验打磨（进行中）

- 甄嬛传角色扩量与字段校对
- 联想体验（拼音等，可选）
- 题包校验（启动时 schema/实体一致性检查加强）
- 可选：题包热加载

✅ 验收：甄嬛传可稳定玩、角色量够一局趣味

### M2 — 更多题包 / 半自动生产

- 用 [08 提示词](./08_猜人物场景_题包生成提示词.md) 批量生成新 pack
- 再做一个正式题材（历史或体育等）
- 仍不强制上 PG

### M3 — 对战（规划）

- Redis 对局状态 + REST 短轮询（[05](./05_对战模式设计.md)）
- 1v1 房间

### M4 — 持久化与上云（规划）

- PostgreSQL：账号、战绩、可选从 pack 导入
- 部署与 HTTPS

### M5 — 小程序（规划）

- 接口契约尽量不动，前端换皮

## 7.2 风险（摘要）

| 风险 | 缓解 |
|------|------|
| 题包手改易不一致 | schema 校验 + 小规模单测 + Code Review |
| 版权 / 数据合规（LoL Demo） | Demo 仅本地验证，正式题材用自有整理数据 |
| 过早上运营后台 | 坚持 ADR-007，玩法跑通再考虑 |
| 对战复杂度 | 严格按 ADR-006，不上 WS 直到必要 |

## 7.3 立项门槛（回顾）

当前已满足「可玩的单人闭环」。下一门槛：正式题材内容量与校对质量，而非基础设施堆叠。
