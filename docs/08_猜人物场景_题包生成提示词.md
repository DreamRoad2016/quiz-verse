# 猜人物场景 — 题包生成提示词

> **用途**：复制到大模型，为某一场景生成 **题包文件**（`pack.yaml` / `schema.yaml` / `entities.json`）。  
> **维护**：通用规则少改；每开新场景只填 **§6 本次场景自定义**。

---

## 0. 角色设定

你是熟悉游戏化产品与 JSON/YAML 内容设计的工程师。请为指定「猜人物」场景输出可直接放入仓库的题包草稿。

输出使用 **中文说明** + **完整可复制的三个文件正文**。

必须包含：

1. 设计摘要（场景边界、列为什么这样选）  
2. `pack.yaml`  
3. `schema.yaml`（含 columns 与 enums）  
4. `entities.json`（至少 8～15 条有辨识度的样例；可标注「待扩充」）  
5. 风险：剧透、版权、枚举遗漏、connections 悬空 id

不要写 Java/SQL，除非自定义区要求。

---

## 1. 产品规则（必须遵守）

- 类 Wordle：限次猜测，服务端按列返回绿 / 黄 / 灰（数值可 ↑↓）
- **比对在服务端**；客户端只有 id + name + aliases
- 内容形态是 **题包目录**，不是数据库表

列类型仅允许：

| type | 含义 |
|------|------|
| identity | 猜中同一人才绿 |
| exact | 单值全等 |
| set | 集合全等绿 / 相交黄 |
| number | 数值；可 nearThreshold |

实体形状：

```json
{ "id": "...", "name": "...", "aliases": [], "attrs": { } }
```

`attrs` 的键必须对应 schema 中列的 `attr`（默认同 `key`）。

---

## 2. 命名建议

- `packId`：小写 + 下划线，稳定可 URL，如 `zhenhuan_2011`
- 实体 `id`：包内唯一，如 `zh_huan`
- 剧情类多值优先存 **key**，展示名放 `enums`

---

## 3. 输出格式要求

请按以下顺序输出，代码块语言标记分别为 `yaml` / `yaml` / `json`：

### pack.yaml
### schema.yaml
### entities.json

---

## 4. 质量自检（生成后自查）

- [ ] 每个 entity 有唯一 id、非空 name  
- [ ] attrs 枚举值落在 schema enums  
- [ ] set 字段为数组  
- [ ] connections（若有）指向本包存在的 id  
- [ ] 列数适中（一般 5～12），适合一屏表格
