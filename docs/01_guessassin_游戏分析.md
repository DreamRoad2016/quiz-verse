# 1. 标杆游戏分析：guessassin.xyz / Guess the Pro

> 调研日期：2026-04-29，URL：https://guessassin.xyz/#/game

## 1.1 一图概览

- 类型：英雄联盟（LoL）职业选手猜谜游戏，**LoLdle 范式**的中文站。
- 核心循环：系统每日（或每局）随机选定一名"目标选手"，玩家在限定次数内通过输入选手名进行猜测，每次提交后表格逐属性显示匹配程度（颜色编码），玩家根据反馈逐步逼近目标。
- 失败/胜利：在猜测次数耗尽前命中即胜，否则系统揭晓答案。

## 1.3 配置项（"游戏设置"弹窗）

| 配置 | 取值 |
| --- | --- |
| 游戏模式 | **经典模式** / **时间陷阱**（带计时挑战） |
| 猜测次数（经典模式） | 7 / 8（默认） / 9 / 10 |
| 题库 | **中韩题库（默认）** / **全球题库（困难）** |

公告还提到了"时间陷阱"是新增模式，"解说杯题库"已迁移到**创意工坊**与**微信小程序**——说明它们已经在做 UGC 题库与小程序版本，方向和我们规划一致。

## 1.4 后端接口与数据加载方式

### 1.4.1 关键接口

观察到游戏只调用一个数据接口：

```http
GET https://guessassin.xyz/api/players/CN_KR  → 200 OK, ~120 KB JSON
```

- `CN_KR` 对应"中韩题库"，"全球题库"对应另一个 region code。
- 一次性把**整个题库**拉到前端，**比对全在客户端完成**。
- 没有"提交一次猜测、服务端返回比对结果"的 round-trip——答题逻辑暴露在 JS 里。

## 1.6 比对算法（逆向推断）

每次猜测后，针对每个属性按字段类型计算颜色：

### 1.6.1 单值字符串 / 枚举（战队、出生地、是否退役）

- 完全相等 → **绿色**
- 不等 → **灰色**

### 1.6.2 多值集合（历史战队、联赛、职位、擅长英雄）

设猜测集合为 G，答案集合为 A：

- `G == A`（全等） → **绿色**
- `G ∩ A ≠ ∅` 且不全等 → **黄色**（部分命中），UI 还会单独高亮命中的元素 chip
- `G ∩ A == ∅` → **灰色**

> 实测：Uzi 的"擅长英雄"显示 `[薇恩, 卢锡安, 卡莎, 霞]`，其中 `卢锡安` 单独标深色块——表示**目标选手的擅长英雄列表里也包含卢锡安**，但其他不重合，所以整体黄底+局部高亮。

### 1.6.3 数值（年龄、世界赛、冠军）

- 相等 → **绿色**
- 不等 → **灰色 + ↑/↓ 箭头**（指示目标值在猜测值的更大/更小方向）
- 还可能有"接近时变黄"的近似规则（很多 LoLdle 类游戏会做"差距 ≤ 阈值"时变黄），从 UI 截图看至少箭头是必有的。

### 1.6.4 ID/姓名

- 完全命中即游戏胜利。
- 未命中显示灰色仅作展示（理论上可以隐藏该列，guessassin 选择保留以方便玩家回看历史猜测）。

## 1.7 输入交互

- 输入框自动联想，根据 `gameId` 与 `realName` 双重检索（输入英文/中文/罗马音都能联想）。
- 输入正确才能点击"猜一下"，否则按钮禁用——确保提交内容必在题库中。

## 1.8 反作弊与公平性

- 由于题库一次拉到前端，**懂技术的玩家可以从内存里直接读出当日答案**——这是 LoLdle 类游戏的通病。
- 该站没有做服务端校验（GET 接口是只读全量数据，没有 POST /guess 之类）。
- 但其受众主要是娱乐玩家，对作弊并不强约束。

## 1.9 可借鉴 / 需要改进的地方

| 点 | 评价 | 我们的处理 |
| --- | --- | --- |
| 一次性全量数据下发 | **可接受**（281 条 ~120KB），加载快、纯前端比对 | 题库小时沿用；上千条以上需考虑接口分页或服务端比对 |
| 配色清晰、表格历史展示 | 体验好 | 全盘借鉴 |
| 多模式（经典/时间陷阱） | 玩法层次丰富 | V2 引入计时模式 |
| 题库可切换（中韩/全球） | 关键扩展点 | 我们直接做成"标签筛选" |
| 没有对战 / 排行榜 | 弱社交 | 我们 V2/V3 重点做对战 |
| 客户端比对易作弊 | 单机娱乐尚可，竞技场景不行 | **对战模式必须服务端裁判** |
| UGC（创意工坊） | 已被验证有需求 | 排到 V3 |

## 1.10 一句话结论

`guessassin.xyz` 把 LoLdle 范式做得很完整，**核心算法 = 11 个属性按"单值/多值集合/数值"三类分别比对 + 颜色编码**，这套规则可以**完整复用**到任意"人物 + 多属性"题材，是我们设计自己产品时直接的算法蓝图。
